Simple flow without email/phone verification

Run this flow here
  • Users sign up with their email address. Phone number is optional.
  • No verification of email address or phone number is done.
  • Users are allowed to sign in with their email address without any verification.
  • Auth Tokens are persisted as cookies which are used to remember the user across page refreshes and tab switches.
  • Phone number, Name and password are displayed after the user signs in.
  • If unauthenticated users try to access a secure url (like /f1/dashboard, they are redirected to the corresponding sign in page)
  • Sign-up/Sign-in with Google is also enabled
